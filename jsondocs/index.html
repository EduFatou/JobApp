Resumen de JobApp

La aplicación es una plataforma para gestionar ofertas de proyectos freelance, donde usuarios y administradores pueden interactuar con distintas funcionalidades relacionadas con los anuncios, usuarios y favoritos. Los usuarios deben identificarse para acceder a estas características.

A continuación se describe la estructura de la aplicación:

  • config: Contiene los archivos de configuración para las bases de datos MongoDB y PostgreSQL.
  • controllers: Incluye los controladores que manejan la lógica de las solicitudes HTTP.
  • middlewares: Contiene los middlewares utilizados en la aplicación.
  • models: Define los modelos de datos para la aplicación.
  • public: Contiene los archivos estáticos como CSS y JS.
  • queries: Incluye los archivos con las consultas SQL para favoritos y usuarios.
  • routes: Define las rutas de la aplicación.
  • services: Contiene los servicios que encapsulan la lógica de negocio.
  • utils: Incluye utilidades como el scraper.
  • validators: Contiene los validadores para las diferentes entidades.
  • views: Contiene las vistas en formato Pug para las distintas páginas de la aplicación.

Archivos en root:

  • index.js: Archivo principal de la aplicación.
  • .env: Archivo de configuración de variables de entorno.
  • jsdoc.json: Configuración para la generación de documentación con JSDoc.
  • package.json: Archivos de configuración de npm.
  • queries.sql: Archivo con consultas SQL.
  • readme.md: Archivo de documentación del proyecto.
